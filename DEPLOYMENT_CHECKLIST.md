# Deployment Checklist - Jarvis AI Web Application

Use this checklist to ensure your application is ready for production deployment.

## Pre-Deployment Phase

### Code Repository
- [ ] All code committed to Git
- [ ] `.env` file NOT committed (check `.gitignore`)
- [ ] Sensitive keys removed from code
- [ ] No hardcoded credentials
- [ ] Latest code pushed to `main` branch

### Environment Setup
- [ ] `requirements.txt` updated with all dependencies
- [ ] `Procfile` created for web servers
- [ ] `runtime.txt` specifies Python 3.10+
- [ ] `.env.example` template created
- [ ] `Dockerfile` created (if using Docker)
- [ ] `docker-compose.yml` created (if using Docker)

### Testing
- [ ] Tested locally with `DEBUG=False`
- [ ] All migrations applied: `python manage.py migrate`
- [ ] Static files collected: `python manage.py collectstatic --noinput`
- [ ] Superuser account created: `python manage.py createsuperuser`
- [ ] All API endpoints tested
- [ ] Form submissions tested
- [ ] File uploads tested
- [ ] Web UI loads correctly
- [ ] No error logs in console

### Django Configuration
- [ ] `SECRET_KEY` changed from default
- [ ] `DEBUG = False` in production
- [ ] `ALLOWED_HOSTS` configured with your domain
- [ ] `DATABASES` configured for PostgreSQL (not SQLite)
- [ ] `STATIC_ROOT` and `STATIC_URL` configured
- [ ] `MEDIA_ROOT` and `MEDIA_URL` configured (if needed)
- [ ] `CORS_ALLOWED_ORIGINS` restricted appropriately
- [ ] Security middleware enabled
- [ ] Error logging configured
- [ ] Email configuration set (for error notifications)

### Security
- [ ] `SECURE_SSL_REDIRECT = True`
- [ ] `SESSION_COOKIE_SECURE = True`
- [ ] `SESSION_COOKIE_HTTPONLY = True`
- [ ] `CSRF_COOKIE_SECURE = True`
- [ ] `CSRF_COOKIE_HTTPONLY = True`
- [ ] Security headers configured
- [ ] CORS origins restricted
- [ ] Rate limiting implemented (optional)
- [ ] SQL injection protection enabled
- [ ] XSS protection enabled

### External Services
- [ ] Gemini API key obtained
- [ ] Gemini API key stored in .env (not code)
- [ ] API rate limits understood
- [ ] API quotas verified for free tier (if applicable)
- [ ] Other third-party services configured

### Database
- [ ] Database service created (PostgreSQL)
- [ ] Database credentials secure
- [ ] Database backup scheduled
- [ ] Database connection tested
- [ ] Migrations are idempotent
- [ ] Database size monitored (if applicable)

### Static Files & Media
- [ ] Static files collected and verified
- [ ] CSS files loading correctly
- [ ] JavaScript files loading correctly
- [ ] Images displaying correctly
- [ ] CDN configured (optional)
- [ ] File permissions correct

### Logging and Monitoring
- [ ] Logging configured for production
- [ ] Error logging emails configured
- [ ] Access logs configured
- [ ] Performance monitoring set up
- [ ] Uptime monitoring configured
- [ ] Alert notifications configured

### Documentation
- [ ] Admin procedures documented
- [ ] Deployment steps documented
- [ ] Rollback procedure documented
- [ ] Recovery procedures documented
- [ ] Maintenance schedule documented

## Rollback Procedure

If deployment fails or causes issues:

### Immediate Actions
- [ ] Stop the failed deployment
- [ ] Revert to previous working version
- [ ] Restore database from backup
- [ ] Clear application cache
- [ ] Restart application

### Revert Code
```bash
# Git revert
git revert <failed-commit>
git push origin main

# Or checkout previous version
git checkout <previous-tag>
git push origin main --force  # USE WITH CAUTION
```

### Restore Database
```bash
# From backup
psql DATABASE_URL < backup.sql
```

### Restart Services
```bash
# Docker
docker-compose restart

# Heroku
heroku restart

# Render/Railway
Trigger manual deploy of previous version
```
